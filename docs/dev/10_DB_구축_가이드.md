# 10. DB 구축 가이드

AICRM 백엔드 DB(PostgreSQL)를 처음부터 세우는 절차. 세 단계로 나뉜다.

| 단계 | 대상 | 도구 | 파일/명령 |
| --- | --- | --- | --- |
| ① Role·DB 생성 | `aicrm` 계정, `aicrm_dev` DB | psql (슈퍼유저) | `backend/db/init.sql` |
| ② 스키마 생성 | 테이블·확장·제약 | Prisma | `prisma migrate deploy` |
| ③ 초기 데이터 | 권한·시드 계정 등 | Prisma seed | `npm run prisma:seed` |

> ①만 Prisma 밖의 수작업이고, ②③은 코드/마이그레이션이 이미 정의하고 있다.

## 사전 준비

- PostgreSQL 13 이상 (btree_gist 확장을 DB 소유자가 생성하려면 13+ 필요)
- `backend/.env` 준비: `cp backend/.env.example backend/.env` 후 값 확인
  - 기준값: `DATABASE_URL="postgresql://aicrm:aicrm@localhost:5432/aicrm_dev"`

## ① Role · Database 생성

```bash
psql -U postgres -h localhost -f backend/db/init.sql
```

- `aicrm` role(로그인 가능)과 `aicrm_dev` 데이터베이스를 생성한다.
- 재실행해도 안전하다(이미 있으면 건너뜀).
- 운영 배포 시 `init.sql`의 비밀번호는 반드시 교체할 것.

## ② 스키마 생성 (마이그레이션 적용)

```bash
cd backend
npx prisma migrate deploy      # 운영/스테이징: 기존 마이그레이션만 적용
# 또는 개발 중 스키마를 바꾸며 작업할 때:
# npx prisma migrate dev
```

- 테이블 전체와 함께 `btree_gist` 확장 및 `rental_allocations` 기간 중복 방지
  `EXCLUDE` 제약이 마이그레이션 SQL 안에서 생성된다.

## ③ 초기 데이터 시드

```bash
cd backend
npm run prisma:seed            # 권한 정의 + 기본 계정 등 (upsert, 재실행 안전)
# 데모 데이터가 더 필요하면:
# npm run seed:demo
# npm run seed:more
```

## 전체 초기화 요약 (로컬 처음 세팅)

```bash
cp backend/.env.example backend/.env
psql -U postgres -h localhost -f backend/db/init.sql
cd backend
npm install
npx prisma migrate deploy
npm run prisma:seed
```

## 참고

- 로컬 개발 DB는 `aicrm_dev`. (구 `aicrm` DB는 이전 프로토타입이므로 건드리지 않는다.)
- DB를 완전히 비우고 다시 만들려면 `npx prisma migrate reset` (마이그레이션 재적용 +
  시드 자동 실행). 단, 해당 DB의 모든 데이터가 삭제되므로 로컬에서만 사용할 것.
